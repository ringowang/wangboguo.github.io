Setup

Install Git, Ruby(using rvm or rbenv), RubyGems, Bundler, PostgreSQL, Redis, nodejs, npm, bower, webpack, webpack-dev-server, imagemagick, ghostscript)

tips:能用brew装的不要用其他方法。

  • Git应该自己装了的吧
  • Ruby & RubyGems & Bundler : 参考 https://ruby-china.org/wiki/install_ruby_guide
  • PostgreSQL: $brew install postgresql
    • 跑完后执行 $brew services start postgresql
  • Redis: $brew install redis (不能按照官网的方式装)
  • nodejs: & npm: $brew install node
  • npm 换源：$npm config set registry https://registry.npm.taobao.org
  • bower：$npm install -g bower
  • webpack: $npm install webpack -g
  • ImageMagick: $brew install ImageMagick
  • ghostscript: $ brew install ghostscript

Test

  • Install Qt (note “capybara-webkit does not support Qt versions greater than 5.5”)：

    • OS X 10.11 or earlier vision $brew tap homebrew/versions,$brew install qt55, then $brew link –force qt55

    • macOS Sierra, see “https://github.com/thoughtbot/capybara-webkit/wiki/Installing-Qt-and-compiling-capybara-webkit#macos-sierra-1012”

  • Download chromedriver and move it to /usr/local/bin,chromedriver
    Git checkout from the repository

  • $npm install cnpm -g –registry=http://registry.npm.taobao.org
    run

  • $sh scripts/test.sh

Install dependency

bundle install

  1. 换成rubychina的源：

    $gem sources --add https://gems.ruby-china.org/ --remove https://rubygems.org/

  2. install ruby gems: bundle install

    问题1：The path /Users/wangboguo/Work/tee_web/tee_core does not exist.

    解决方案：

    $ln -s ~/Work/tee_core/ ~/Work/tee\_web（需要换成自己的路径我的Work是在根目录下）

    坑：必须要用绝对链接，这样ln -s ./tee_core ./tee_web 不行

    问题2:Make sure that gem install libv8 -v '3.16.14.13' succeeds before bundling.

    解决: 

    $gem install libv8 -v '3.16.14.13' -- --with-system-v8

    这一步可能又出现问题，解决方案如下：

       $gem uninstall libv8 
$gem install therubyracer -v '0.12.2' 
$gem install libv8 -v '3.16.14.13'  -- --with-system-v8

    问题3：Make sure that gem install capybara-webkit -v '1.11.1' succeeds before bundling.

    解决方案：

    $gem install capybara-webkit -v '1.11.1'

    问题4： sh: qmake: command not found

    解决方案： 照着第二步教程搞好QT应该就没问题

    问题5: 在tee_admin bundle i的时候，出现Make sure that gem install capybara-webkit -v '1.7.1' succeeds before bundling.
    然后gem install capybara-webkit -v ‘1.7.1’时有新问题：sh: qmake: command not found

    解决方案： 

    在根目录打开bash_profile: $vi .bash_profile 或者 $atom .bash_profile
添加一行: export $PATH="/Users/wangboguo/Qt5.5.1/5.5/clang_64/bin:$PATH";
这样就能找到qmake了之后再在admin里面bundle i就没问题了

Prepare Database (see project_dir/config/database.yml)

  • create database user tee and tee_tester

    • 下载pgadmin3: https://www.pgadmin.org/download/macos.php

  • create database tee_development and tee_test

  • if brew into redis run $brew services start redis background
    service

  • $rake db:seed

  • 导入两个数据库用pgAdmin3

  • $RAILS_ENV=test rake db:migrate

  • $RAILS_ENV=test rake db:seed
